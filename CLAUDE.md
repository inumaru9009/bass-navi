# CLAUDE.md — Bass Navi 開発規約・教訓

## 基本方針

- 大きな構造変更は必ず **Writeツールで完全上書き** を明示すること
- Edit ツールは小規模修正のみ。複数箇所にまたがる変更は Write を使う
- 長いセッションでは定期的に新チャットへ移行すること

---

## 指示書テンプレート

Claude.ai で方針を決めてから指示書を渡すこと。指示書には以下を含める：

```
## 修正内容
（何をどう変えるか）

## 変更しないこと
（確認済みの判断・触らない箇所）

## 再現手順
（曲名・コード名・具体的な操作手順）

## 確認観点
（修正後に確認するケース一覧）
```

## 過去に確認済みの判断（変更禁止）

### 正規表現
- `replace(/\s+/g, ' ')` は **使わない**
- 代わりに `replace(/[ \t\n\r]+/g, ' ')` を使う
- 理由: テンプレートリテラル内で `\s` が壊れ `s` を削除するバグが発生した

### DOM取得
- 歌詞取得は `col.innerText || col.textContent` を使う
- `textContent` のみだと CSS pseudo-element でレンダリングされた文字が取れない

### 歌詞の空col処理
- 空の `.col` はスペース `" "` に変換してから結合する
- 理由: U-FRETの `.col` は1文字ずつ分割されており、空文字が単語区切りを表す

---

## buildAllPositions の設計判断

- ルートが複数弦にある場合、**最低フレットのルート1つだけ残す**
- 理由: 中央値ベースの外れ値フィルターは数学的に除外できないケースがあった
- `OPEN_STRINGS` は C=0基準（E=4, A=9, D=2, G=7）— E=0基準と混同しないこと
- `beat` は 0始まり

## デバッグのアプローチ

- 「表示がおかしい」系の問題は、まず実データをコンソールで確認してから修正する
- 問題は「再現手順（曲名・コード名・具体操作）」とセットで渡すと一発解決しやすい
- 確認は網羅的に行う（例: 指板図は17コード以上のスクリーンショットで確認）
